# 0.60.0 — Release Notes

## Summary

This release consolidates the `ai-new` bootstrap workflow: it resolves all
defects identified during the `kaos-control` bootstrap trial run, hardens
profile validation against generated-scaffold errors, adds directory-mismatch
detection for mid-session agent renames, and introduces first-class model/effort/
approval tuning for bootstrap agents including a new `claude` agent entry.

---

## Defects resolved

### From kaos-control bootstrap trial run (D-001 – D-009)

| ID | Area | Description |
|----|------|-------------|
| D-001 | Prompt conventions | Prompt used obsolete `CODEX_JAILS_DIR`; updated to `AI_PODMAN_JAILS_DIR` throughout |
| D-002 | Prompt conventions | Prompt referenced obsolete `CODEX_BIN` launcher convention; updated to `AI_PODMAN_BIN` |
| D-003 | Quality gate | Supervisor reported failure after successful image commit; `quality-gate-inconsistent` status added |
| D-004 | Result serialisation | `error_summary` captured successful output; fixed to reject success-tail content in failed results |
| D-005 | Repair workflow | Repair budget exhausted by supervisor-side false negatives; contradictory results no longer count against the budget |
| D-006 | Scaffold generation | `PODMAN_BUILDER.md` generator now extracts packages, ports, ENV vars, and mounts from real files instead of emitting deferrals |
| D-007 | Validation reporting | `static_check_status` was always `skipped`; field now reflects real check outcome |
| D-008 | Bootstrap ergonomics | Local file inspection in the bootstrap container failed under Bubblewrap; shell wrappers no longer intercept simple reads (abandoned — addressed via prompt guidance instead) |
| D-009 | Protocol design | Session state remained `quality-gate-failed` even when a valid image existed; `quality-gate-inconsistent` terminal state added |

### Additional defects (post-trial triage, 2026-06-28)

| Lineage | Area | Description |
|---------|------|-------------|
| `install-sh-omits-version-file` | Installer | `install.sh` did not copy `VERSION` to `$AI_PODMAN_JAILS_DIR`, causing `ai-pod-doctor` to exit with "no version marker" on every run |
| `ai-launch-no-close-match-suggestion` | CLI UX | `ai-launch <name>` failed with no hint when a close-match profile existed (e.g. typed `kaos-control`, profile is `kaos-control-dev`) |
| `bootstrap-extra-env-wrong-format` | Scaffold generation | Bootstrap generated bare `KEY=VALUE` in `EXTRA_ENV`/`EXTRA_VOLUMES`; validator requires alternating `-e`/`-v` + value pairs; prompt and template now document correct format |
| `bootstrap-agent-rename-breaks-host-directory` | Bootstrap | Agent suggested renaming the project slug mid-session, producing a scaffold referencing a non-existent host directory; prompt now prohibits mid-session renames; `validate_launchability_contract` detects mismatch and emits the corrective `mv` command |

## New features

### Agent model, effort, and approval tuning

Three optional registry keys added to the agent registry schema:

| Key | Effect |
|-----|--------|
| `AGENT_MODEL` | Passes `--model <value>` at agent launch |
| `AGENT_EFFORT` | Passes effort-level flag at agent launch (agent-specific) |
| `AGENT_APPROVAL` | Passes approval/permission policy flag at agent launch |

Bootstrap agents now target mid-tier models at medium effort with reduced
approval interruptions inside the sandboxed container:

| Agent | Model | Effort | Approval |
|-------|-------|--------|----------|
| `codex` | `gpt-5.4` | `medium` (`--reasoning-effort`) | `full-auto` (`--approval-policy`) |
| `claude` | `claude-sonnet-4-6` | — | `skip-permissions` (`--dangerously-skip-permissions`) |
| `gemini` | — | — | — |

### Default container bashrc (QOL)

A framework-managed `config/bashrc.default` is now seeded into the image build
context at scaffold time and baked into every new container image at
`/etc/ai-podbuilder/bashrc`. The container shell sources it on startup instead
of the workspace `.bashrc`, so the environment is set up without any
per-deployment manual copy.

Includes: `ll`/`la`/`l`/`grep` aliases, coloured `ls`, custom PS1
(`user@ai-podbuilder:dir$`), `set -o vi`, and `.bashrc.d/*.sh` drop-in
loading. User or project addins go in `~/.bashrc.d/` inside the container.
The workspace `.bashrc` is still sourced at the end for per-project
customisation.

The bootstrap prompt now instructs agents to `COPY bashrc.default /etc/ai-podbuilder/bashrc`
in the generated Containerfile; the durable Containerfile template already
includes this step.

### New `claude` agent entry

`config/agents.d/claude.env` registers Claude Code (`@anthropic-ai/claude-code`)
as a first-class `ai-new` bootstrap agent. Use with:

```bash
ai-new myproject --agent claude
```
