# 0.61.0 — Release Notes

## Summary

This release streamlines the `ai-new` bootstrap interview from 21 questions to
8, adds a standard QoL toolset and per-language linting to every generated
Containerfile, fixes the shell prompt hostname, isolates per-project agent
configs in `state/home/`, and hardens the bootstrap container with `bubblewrap`
and `hadolint`.

---

## Defects resolved

| Lineage | Area | Description |
|---------|------|-------------|
| `ps1-hardcoded-hostname` | UX | PS1 hardcoded `ai-podbuilder`; changed to `\h` so the prompt reflects the actual container hostname |
| `hadolint-not-in-bin` | Install | `hadolint` downloaded to `lib/` but not symlinked into `bin/`; not on `$PATH` for direct invocation |
| `bootstrap-missing-bubblewrap-hadolint` | Bootstrap | Bootstrap container lacked `bubblewrap` (sandbox execution) and `hadolint` (Containerfile linting) |
| `codex-model-name-recased-uppercase` | Agents | `AGENT_MODEL` in `codex.env` re-set to uppercase `GPT-5.4` by a prior fix; codex CLI requires lowercase `gpt-5.4` |

---

## New features

### Bootstrap interview streamlined (21 → 8 questions)

The interview checklist is reduced and restructured:

- Redundant questions merged (purpose + role → one question)
- Framework-default questions dropped (rootless, Podman-only, helper scripts,
  README — always generated)
- Agent runtime now lists explicit options: `codex`, `claude`, `gemini`,
  `other`, `none`
- **Repo-review shortcut**: if the project involves coding and the user has an
  existing repo, the agent reads it and infers the language stack, package
  managers, and tooling without asking
- **Env vars and secrets are passive**: `EXTRA_ENV=()` by default; only
  populated if the user volunteers values during the interview
- Gated questions for mounts and persistent state default to empty arrays and
  only continue if the user has something to add

### Standard QoL toolset baseline

Every generated Containerfile now includes regardless of project type:
`git`, `gh`, `nano`, `neovim`, `ripgrep`, `fzf`, `lazygit`, `pnpm`, `uv`.
Install methods documented per distro (Fedora/Ubuntu). Not asked during
interview — just always included.

### Language tooling inferred automatically

Standard linters, formatters, and build tools included per detected language
without any interview question:

| Language | Auto-included tooling |
|----------|-----------------------|
| Python | ruff, black, mypy, pylint, make |
| Node / TypeScript | eslint, prettier, tsc, make |
| Go | golangci-lint, gofmt (built-in), make |
| Rust | rustfmt + clippy (via rustup), make |
| Ruby | rubocop, make, rake |
| Java | checkstyle, spotbugs, maven/gradle |
| Shell / Bash | shellcheck, make |
| C / C++ | clang-format, cppcheck, make, cmake, ninja |

`make` and `shellcheck` always included regardless of primary language.

### Per-container hostname

- `bashrc.default` PS1 now uses `\h` (actual hostname) instead of `ai-podbuilder`
- Bootstrap container launched with `--hostname "ai-new-<slug>"`
- Durable container launched with `--hostname "${CONTAINER_HOSTNAME:-${CONTAINER_NAME}}"`
- `CONTAINER_HOSTNAME` added as optional `profile.env` field
- Bootstrap interview asks for hostname; suggests project slug as default

### Per-project agent config isolation

`scaffold_layout` now:
- Creates `state/home/` as part of the canonical project structure
- Seeds it with project-local copies of `.codex`, `.claude`, `.gemini`,
  `.config/gh` from the host `$HOME` at creation time

Each project gets its own isolated agent credentials. Changes inside one
project's container do not affect others. Existing profiles with EXTRA_VOLUMES
entries for these dirs continue to work unchanged.
